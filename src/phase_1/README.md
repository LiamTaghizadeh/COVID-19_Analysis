# مبدل JSON به CSV

یک ابزار سبک و شیءگرا با پایتون برای تبدیل داده‌های JSON به فرمت CSV. این ابزار برای سادگی و قابلیت اطمینان طراحی شده است و هر دو نوع شیء JSON تکی و آرایه‌ای را پشتیبانی می‌کند. همچنین ساختارهای تو در تو را به صورت رشته‌های JSON در ستون‌ها ذخیره می‌کند تا اطلاعات از دست نرود.

---

## 📌 منبع داده

داده‌های JSON استفاده‌شده در این پروژه از صفحه‌ی آمار لحظه‌ای بیماری کووید-۱۹ در وب‌سایت **سیب مگزین** گرفته شده است:

> **[seebmagazine.com/coronavirus-covid-19-info/](https://seebmagazine.com/coronavirus-covid-19-info/)**

این صفحه اطلاعات به‌روز و کشور-به-کشور را شامل موارد زیر ارائه می‌دهد:  
تعداد کل مبتلایان، موارد جدید، فوتی‌ها، بهبودیافتگان، موارد فعال، تعداد آزمایش‌ها، جمعیت و شاخص‌های نسبی. ساختار JSON استفاده‌شده در این ابزار دقیقاً منطبق با داده‌های همان صفحه است.

---

## ✨ ویژگی‌ها

- **طراحی شیءگرا** – کد تمیز، ماژولار و به‌راحتی قابل توسعه.
- **پشتیبانی از هر دو نوع JSON** – کار با یک شیء تکی یا یک آرایه از اشیاء.
- **تشخیص خودکار فیلدها** – همه‌ی کلیدهای سطح اول از تمام رکوردها جمع‌آوری می‌شوند تا هدرهای CSV کامل شوند.
- **پشتیبانی از مقادیر تو در تو** – اشیاء و آرایه‌های تو در تو به رشته‌های JSON تبدیل می‌شوند که یکپارچگی داده‌ها حفظ شود.
- **پذیرش آرگومان خط فرمان** – می‌توانید مسیر فایل ورودی و خروجی را مستقیم به اسکریپت بدهید.
- **حالت تعاملی** – اگر آرگومانی ارسال نشود، از کاربر مسیر فایل‌ها پرسیده می‌شود.
- **کدگذاری UTF-8 با BOM** – برای سازگاری با اکسل و نرم‌افزارهای صفحه‌گسترده.

## 📊 توضیح ساختار داده‌های JSON

داده‌های دریافتی از وب‌سایت سیب مگزین شامل اطلاعات همه‌گیرشناسی (اپیدمیولوژی) برای هر کشور است. در زیر هر فیلد و معنی آن توضیح داده شده است:

| فیلد | نوع | توضیح |
|------|------|-------|
| `updated` | عدد (Timestamp) | زمان آخرین به‌روزرسانی داده به میلی‌ثانیه از ۱۹۷۰ |
| `country` | رشته | نام کشور |
| `countryInfo` | شیء | اطلاعات تکمیلی کشور شامل `_id`، `iso2`، `iso3`، مختصات (`lat`، `long`) و آدرس پرچم |
| `cases` | عدد | تعداد کل موارد ابتلا تاکنون |
| `todayCases` | عدد | موارد جدید ثبت‌شده در امروز |
| `deaths` | عدد | تعداد کل فوتی‌ها |
| `todayDeaths` | عدد | فوتی‌های جدید امروز |
| `recovered` | عدد | تعداد کل بهبودیافتگان |
| `todayRecovered` | عدد | بهبودیافتگان جدید امروز |
| `active` | عدد | موارد فعال (در حال درمان) |
| `critical` | عدد | موارد بحرانی (بستری در بخش مراقبت‌های ویژه) |
| `casesPerOneMillion` | عدد | تعداد مبتلایان به ازای هر یک میلیون نفر جمعیت |
| `deathsPerOneMillion` | عدد | تعداد فوتی‌ها به ازای هر یک میلیون نفر |
| `tests` | عدد | تعداد کل آزمایش‌های انجام‌شده |
| `testsPerOneMillion` | عدد | تعداد آزمایش به ازای هر یک میلیون نفر |
| `population` | عدد | جمعیت کشور |
| `continent` | رشته | قاره‌ی مربوطه |
| `oneCasePerPeople` | عدد | به ازای هر چند نفر یک مورد ابتلا (تقریباً) |
| `oneDeathPerPeople` | عدد | به ازای هر چند نفر یک فوتی (تقریباً) |
| `oneTestPerPeople` | عدد | به ازای هر چند نفر یک آزمایش (تقریباً) |
| `activePerOneMillion` | عدد | موارد فعال به ازای هر یک میلیون نفر |
| `recoveredPerOneMillion` | عدد | بهبودیافتگان به ازای هر یک میلیون نفر |
| `criticalPerOneMillion` | عدد | موارد بحرانی به ازای هر یک میلیون نفر |

> **نکته:** فیلد `countryInfo` به صورت یک رشته‌ی JSON در خروجی CSV ذخیره می‌شود تا بتوانید در صورت نیاز آن را تجزیه کنید.

## 🧪 مثال

**ورودی (`data.json`):**

```json
{
  "updated": 1784464632990,
  "country": "Zimbabwe",
  "cases": 266359,
  "todayCases": 0,
  "deaths": 5740,
  "todayDeaths": 0,
  "recovered": 258888,
  "todayRecovered": 0,
  "active": 1731,
  "critical": 12,
  "casesPerOneMillion": 17373,
  "deathsPerOneMillion": 374,
  "tests": 2525756,
  "testsPerOneMillion": 164744,
  "population": 15331428,
  "continent": "Africa",
  "oneCasePerPeople": 58,
  "oneDeathPerPeople": 2671,
  "oneTestPerPeople": 6,
  "activePerOneMillion": 112.91,
  "recoveredPerOneMillion": 16886.1,
  "criticalPerOneMillion": 0.78,
  "countryInfo": {
    "_id": 716,
    "iso2": "ZW",
    "iso3": "ZWE",

    "long": 30,
```

**اجرای فرمان:**

```bash
python json_to_csv.py data.json result.csv
```

**خروجی (`result.csv`):**

| updated       | country  | cases  | deaths | recovered | active | ... | countryInfo                                                                                                  |
|---------------|----------|--------|--------|-----------|--------|-----|--------------------------------------------------------------------------------------------------------------|
| 1784464632990 | Zimbabwe | 266359 | 5740   | 258888    | 1731   | ... | {"_id":716,"iso2":"ZW","iso3":"ZWE","lat":-20,"long":30,"flag":"https://disease.sh/assets/img/flags/zw.png"} |

(ستون‌های دیگر نیز به همین ترتیب درج می‌شوند.)

## 🧱 ساختار کلاس‌ها

کلاس اصلی `JsonToCsvConverter` متدهای زیر را دارد:

| متد | توضیح |
|-----|-------|
| `__init__(input_path, output_path)` | سازنده، مسیر فایل‌ها را دریافت می‌کند. |
| `load_json()` | فایل JSON را خوانده و پارس می‌کند. |
| `normalize_data()` | اگر داده یک شیء باشد، آن را به یک لیست تبدیل می‌کند تا پردازش یکنواخت شود. |
| `collect_fieldnames()` | همه‌ی کلیدهای سطح اول از تمام رکوردها را جمع‌آوری کرده و مرتب می‌کند. |
| `flatten_value(value)` | اگر مقدار از نوع دیکشنری یا لیست باشد، آن را به رشته‌ی JSON تبدیل می‌کند. |
| `write_csv()` | با استفاده از هدرهای جمع‌آوری‌شده، فایل CSV را می‌نویسد. |
| `convert()` | متد اصلی که همه‌ی مراحل را به ترتیب اجرا می‌کند. |
